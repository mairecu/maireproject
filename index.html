Introduction to programming
Session 1

HTML is the language used for web pages. It's made of text content, markup, references to other documents and links to other pages.

The markup, which defines how the content looks like, has tags, elements and attributes. Those let us reflect what we really want to make out of our web page.

Session 2

In session 2 i've learned the first and more basic ways to create a Structured document with HTML.

First of all, it is important to remember the function of each component of the web page:

- HTML: Structure

- CSS: Style

- JAVASCRIPT: Interactive components

HTML posses syntax and rules. This syntax contents tags that are organized in a Tree like struture. Each HTML tag creates an element in the DOM (Document object model) that the browser uses to display the page.

All components within a web page desing is inside a box. This way is much easy to organized, move and detect every single detail that needs to be modify. This technique is known as Boxify a Desing.

Session 3

In this session i got to understand and work with CSS (Cascading Style Sheets). You can have several CSS files or sheet that wil be referenced in HTML and all of those are gonna be used to find out the final and visible style for your page.

A very important concept to apply is the Inheritance, wichi is the mechnism by wich properties are applied not only to a specified element, but also to its descendants. Inheritance prevents certain properties for being declared over and over again in a style sheet, allowing the software developers to wirte less code. It enhances faster-loading of web pages by users and enables the client to save money on band width and development costs.

Properties that can be inherited

-Color
-Font
-Letter-spacing
-Line-height
-List-style
-Text-align
-Text-index
-Text-transform
-Visivility
-White space
-Word-spacing

Properties that can not be inherited

-Background

-Height and width

-Min. and max. height and width
-Outline
-Overflow

-Position
-Text-decoration
-Vertical-align
-Z-index

Box sizing

HTML are boxes and each box has four components. There are two techniques we can use to help deal with sizing issues:

-Set sizes in terms of percent (%) rather than pixels.

-Set the box-sizing attribute to border-box for every element.

Box positioning

Divs are blocks elements, so by default they take up the entire width of a page. Adding the rule display-flex; to the appropiate CSS will overrride this behavior and let divs appear next to each other.
